# mud

一个基于命令行的单机文字 RPG（MUD 风格）项目。  
玩家可以在地图中移动、与 NPC 对话推进主线、参与回合制战斗、在商店买卖、管理背包与装备，并通过存档继续进度。

## 功能概览

- 回合制战斗：普通攻击、技能、药品、逃跑
- 主线对话：多 NPC 剧情推进与任务切换
- 角色成长：经验、等级、属性与技能点
- 物品系统：武器、防具、药品、掉落与商店交易
- 存档系统：文本存档 `savegame.txt`
- 地图探索：多区域移动与区域事件触发

## 项目结构

- `include/core`：主流程与应用入口相关接口
- `include/systems`：系统层（战斗、对话、商店）
- `include/domain`：领域模型（角色、地图、物品、技能、任务等）
- `include/io`：输入输出与存档相关接口
- `src/core`：主流程实现
- `src/systems`：系统层实现
- `src/domain`：领域模型实现
- `src/io`：IO 层实现
- `legacy`：历史实现备份（不参与当前构建）

## 构建与运行

### 环境要求

- Linux / macOS
- `g++`（支持 C++17）

### 构建

```bash
g++ -std=c++17 -Wall -Wextra -pedantic $(find src -name '*.cpp' | sort) -Iinclude -o mud
```

### 运行

```bash
./mud
```

## 运行与数据文件

- 存档文件：`savegame.txt`
- 若需要重开流程，可删除存档文件后重新运行

## 开发说明

- 主流程入口在 `src/core/Main.cpp`，核心调度在 `src/core/Game.cpp`
- 新玩法优先新增到 `systems` 层，尽量避免把业务逻辑直接塞回 `Game`
- `domain` 层负责状态与规则，`io` 层负责控制台和持久化
- `legacy` 目录仅用于参考，默认不应继续新增逻辑
